## 方案一：多标签仪表盘 — TRAE Prompt

```
# 任务：构建 LLM 逻辑推理评估仪表盘（多标签交互式 Web 应用）

## 项目背景
本项目是一个针对大语言模型（LLM）逻辑推理能力的自动化评估框架。
评估框架将推理链（Chain of Thought）抽象为树形结构，输出三个核心指标：
- Score_Depth（0-100）：推理链有效深度，衡量每一步的逻辑增益
- Score_Breadth（0-100）：假设空间覆盖率，衡量反事实/边缘情况的考量
- Score_Consistency（0-100）：推理链路的自洽性

评估的输入格式为：Model_Prompt + Model_Response
评估的输出格式为：JSON，包含三项分数及推理树结构

## 技术栈要求
- 纯前端单页应用，不依赖后端
- 使用 React + TypeScript
- 图表库：Recharts（雷达图、柱状图）
- 样式：Tailwind CSS
- 所有数据用 Mock 数据硬编码（模拟真实评估输出）

## 方案二：推理树断裂检测器 — TRAE Prompt

```
# 任务：构建 LLM 推理链断裂检测与深度分析器（专项交互式 Web 工具）

## 项目背景
本工具是「大模型逻辑推理广度与深度评估算法」项目的核心可视化组件。
项目将 LLM 的 Chain of Thought（CoT）推理过程抽象为树形结构，并检测其中的「逻辑断裂」节点。

### 核心概念定义
- **逻辑断裂（Break）**：前一推理步骤 St 无法在逻辑上必然推导出下一步骤 St+1，
  或中间缺失了必要的隐含前提。表现为「看似连贯实则瞬移」的推理步骤。
- **Score_Depth**：非冗余逻辑增益的累计值。每一步必须让当前状态更接近目标状态，
  使用信息熵/互信息衡量每步的「逻辑推进力」。断裂节点会降低此分数。
- **Score_Breadth**：有效假设空间的覆盖率。对反事实、边缘情况和竞争性分支的考量。
- **Score_Consistency**：推理链路的自洽性，检测前后矛盾。

## 技术栈要求
- React + TypeScript
- 样式：Tailwind CSS
- 图表：Recharts（仅用于右侧评分条形图）
- SVG 推理树：纯 React 手写渲染，不使用任何树布局库
- 动画：使用 Framer Motion（节点出现动画、选中动画）

## 数据定义

### 类型定义
```typescript

type RiskLevel = '无' | '低' | '中' | '高';

  id: string;
  x: number;
  y: number;
  label: string;
  subLabel: string;
  type: NodeType;
  logicGain: string;
  gainValue: number;       // 纯数值，用于进度条，范围 -0.1 到 0.5
  premiseStatus: PremiseStatus;
  riskLevel: RiskLevel;
  description: string;
  parentIds: string[];
  childIds: string[];
  stepIndex: number;       // 在推理链中的顺序编号（用于动画播放）
}

interface ModelProfile {
  name: string;
  scoreDepth: number;
  scoreBreadth: number;
  scoreConsistency: number;
  breakNodeCount: number;   // 该模型推理此题时的断裂节点数
  redundantNodeCount: number;
  validNodeCount: number;
  totalSteps: number;
}
```

## 界面结构要求（单页，不分 Tab）

### 整体布局（三栏式）

```
┌────────────────────────────────────────────────────────────────────────┐
│  顶部栏：标题 + 题目选择器 + 模型选择器                                  │
├──────────────────────────────┬─────────────────────────────────────────┤
│  左栏（240px）                │  中栏（主内容区，自适应）                  │
│  - 统计摘要卡片               │  推理树 SVG 可视化                        │
│  - 节点类型图例               │                                          │
│  - 检测结果列表               │                                          │
├──────────────────────────────┴─────────────────────────────────────────┤
│  底栏：三维评分面板（水平排列，5个模型的得分对比）                          │
└────────────────────────────────────────────────────────────────────────┘
```

### 顶部栏
- 左侧：标题「推理链断裂检测器」（加粗，18px）+ 副标题「Logical Break Detection & Depth Analysis」（12px，灰色）
- 中间：题目选择器（三个题目的 Radio Button 样式，横向排列，当前题目高亮蓝色，包含难度标签 chip）
- 右侧：模型选择下拉框（5个模型，切换后底栏高亮对应模型数据）
- 右侧还有一个「逐步播放」按钮（▶ 播放 / ■ 停止），点击后节点按 stepIndex 顺序逐个出现，间隔 600ms

### 左栏（固定 240px 宽）

**统计摘要卡片**（当前题目 + 当前模型）：
- 4个小指标格子（2×2）：
  * 总步骤数（动态）
  * 有效节点数（绿色数字）
  * 断裂节点数（红色数字，有断裂时显示警告图标 ⚠）
  * 冗余节点数（灰色数字）

**深度评分分解**：
- 显示「Score_Depth: XX」（大字）
- 下方用小进度条分解：有效步骤贡献 / 断裂扣分 / 冗余无贡献

**节点类型图例**（色块 + 标签）：
- ● 有效推理节点（蓝色）
- ● 逻辑断裂节点（红色，⚠）
- ● 冗余步骤（灰色，虚线边框）
- ● 结论节点（绿色）
- ● 根节点（深灰色）

**检测结果列表**（仅显示 break 类型节点）：
- 每条：节点ID + 简短说明 + 风险等级 chip
- 点击后在树中高亮对应节点

### 中栏（推理树 SVG 主视图）

**工具栏**（树图上方）：
- 「全部展开 / 全部折叠」按钮
- 「缩放重置」按钮（图标：⊙）
- 「仅显示断裂路径」toggle（开启后，非断裂相关节点半透明）
- 「显示增益标注」toggle（开启后，每条连线上显示 logicGain 数值）

**SVG 推理树**：
- 容器：白色背景，1px 灰色边框，rounded-xl，可横向滚动
- SVG 尺寸：根据题目动态调整（Q1: 640×360px, Q2: 640×340px, Q3: 640×340px）
- 节点样式（矩形，宽116px，高46px，圆角8px）：
  * root：#F8FAFC 填充，#64748B 边框（1px）
  * valid：#EFF6FF 填充，#2563EB 边框（1px）
  * break：#FEF2F2 填充，#DC2626 边框（2px），节点左上角红色三角警告标记（SVG polygon）
  * redundant：#F9FAFB 填充，#9CA3AF 虚线边框（1px, stroke-dasharray 4 2）
  * conclusion：#F0FDF4 填充，#059669 边框（2px）
- 节点内文字：
  * 主标签（label）：13px，font-weight 600，颜色根据类型（root深灰/valid深蓝/break深红/redundant灰/conclusion深绿）
  * 副标签（subLabel）：10px，颜色同主标签但 opacity 0.65
- 连接线样式：
  * 普通连线：1px，颜色 #CBD5E1
  * 连向/来自 break 节点的连线：1.5px，颜色 #DC2626，stroke-dasharray 6 3
  * 连向 redundant 节点的连线：0.8px，颜色 #D1D5DB，stroke-dasharray 4 2
  * 所有连线带箭头（marker-end），箭头样式：小三角，6×6px
- 「显示增益标注」开启时，每条连线中点显示一个小 badge（白色背景，灰色边框，10px字体，显示增益值）
- **选中节点**：
  * 边框变粗（3px），出现 4px 的彩色 box-shadow（颜色与节点类型一致）
  * 节点右侧出现一个浮动详情气泡（popover，宽 220px），白色背景，阴影，包含：
    - 节点标题（type badge + label）
    - logicGain（带颜色，正值绿色，负值红色）
    - premiseStatus（chip）
    - riskLevel（chip）
    - description 文字（12px，灰色，多行，line-height 1.6）
    - 底部「×」关闭按钮
- **hover 状态**：节点轻微放大 scale(1.04)，cursor pointer
- **逐步播放模式**：
  * 节点按 stepIndex 顺序，每隔 600ms 出现一个
  * 出现动画：从 opacity:0, scale:0.85 到 opacity:1, scale:1（Framer Motion）
  * 连线跟随子节点一起出现
  * 播放完毕后按钮恢复「▶ 重播」
- **「仅显示断裂路径」模式**：
  * 非断裂路径上的节点和连线 opacity 降至 0.2
  * 断裂节点及其直接父节点、子节点保持完整显示
  * 切换时有 300ms 过渡动画

### 底栏（三维评分对比面板）

**5个模型的评分卡并排**（flex 布局，等宽）：
- 当前选中模型的卡片：白色背景，2px 蓝色边框，轻微上移阴影
- 未选中模型的卡片：浅灰背景（#F8FAFC），1px 灰色边框
- 每张卡片内容：
  * 顶部：模型名（14px 加粗）+ 断裂数 badge（红色 chip，如「2 断裂」）
  * 三条进度条（带标签和数值）：
    - Score_Depth：蓝色 (#2563EB)
    - Score_Breadth：紫色 (#7C3AED)
    - Score_Consistency：绿色 (#059669)
  * 底部：综合分（大字，计算三项均值）
- 卡片点击后切换为「当前选中模型」，树图左侧统计数据随之更新

## 交互联动要求
1. **题目切换**：树图平滑过渡（左右滑动动画，300ms），左栏统计数据更新，底栏分数更新
2. **模型切换**：底栏对应卡片高亮，左栏分数更新；树的结构保持不变（结构由题目决定），但断裂节点的警告角标会根据模型稍有不同（仅 UI 效果，实际数据不变）
3. **节点选中与左栏联动**：点击树节点时，若该节点是 break 类型，左侧「检测结果列表」中对应条目高亮
4. **「仅显示断裂路径」与左侧联动**：开启时，左侧检测结果列表也同步聚焦（非断裂条目灰显）

两个 prompt 的关键差异在于：**方案一** 是宽泛的评估结果汇总展示（4 Tab，适合汇报），**方案二** 是聚焦于单次推理过程的深度解剖工具（逐步播放、断裂路径高亮、节点气泡），二者互补。数据全部硬编码在 prompt 里，TRAE 无需额外数据源即可完整复现。
